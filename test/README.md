# USB Display Control - 测试套件

本目录包含所有测试相关的文件和脚本。

## 测试文件说明

### 自动化测试脚本

| 文件名 | 说明 | 用途 |
|--------|------|------|
| `test_display_control.py` | 主测试脚本 | 完整的自动化测试套件 |
| `test_adb.py` | ADB 测试 | 测试 ADB 连接和设备检测 |
| `test_client.py` | 客户端测试 | 测试 HTTP 客户端功能 |
| `test_displays.py` | 显示器检测测试 | 测试显示器状态检测 |
| `test_monitors.py` | 显示器监控测试 | 测试显示器切换功能 |

### 测试规范

| 文件名 | 说明 |
|--------|------|
| `TEST_SPEC.md` | 产品测试规范文档 |

## 运行测试

### 运行完整测试套件

```bash
cd test
python test_display_control.py
```

### 运行单个测试模块

```bash
# 测试 ADB 功能
python test_adb.py

# 测试显示器检测
python test_displays.py

# 测试显示器切换
python test_monitors.py
```

## 测试覆盖

测试套件覆盖以下功能模块：

1. **服务器基础功能** (TEST-001)
   - 端口监听
   - HTTP 请求处理
   - 响应格式验证

2. **显示器状态检测** (TEST-002)
   - 显示器数量检测
   - 主显示器识别
   - 显示模式判断

3. **显示器切换控制** (TEST-003)
   - 切换到仅第一屏
   - 切换到仅第二屏
   - 切换到扩展模式

4. **Android 端通信** (TEST-004)
   - HTTP 客户端连接
   - 状态获取
   - 命令发送

5. **错误处理** (TEST-005)
   - 服务器不可达
   - 无效命令
   - 超时处理

6. **集成测试** (TEST-006)
   - 完整工作流程
   - 端到端测试

## 测试报告

运行测试后会生成详细的测试报告，包括：
- 测试用例执行情况
- 通过率统计

## 注意事项

1. **运行环境**
   - Windows 10/11
   - Python 3.8+
   - ADB 已安装并配置

2. **测试前准备**
   - 确保 USB 调试已启用
   - 确保显示器驱动正常
   - 关闭其他可能干扰的程序

3. **测试执行**
   - 某些测试需要管理员权限
   - 显示器切换会导致屏幕闪烁
   - 测试过程中不要手动操作显示器

## 贡献指南

添加新测试时，请遵循以下规范：

1. 使用统一的测试框架（unittest）
2. 提供清晰的测试说明
3. 包含预期结果和实际结果
4. 添加适当的日志输出
5. 更新 TEST_SPEC.md 文档
